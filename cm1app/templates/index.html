Optode readings

Fields in a voltage report (left to right)

  • tx_id: # of times XBee Tx is called
  • bad_char_count (# of non-ASCII characters received from sensor so far)
  • ticker: internal 1Hz "clock"
  • last_transmitted (ticker and last_transmitted should be max 20min apart)
  • last_received (ticker and last_received should be max 20min apart)
  • Vcc (should be 3.3V)
  • Vbatt (should be 5V)

Note 1: 3835 optode sends some control characters with every message, so bad_char_count is expected to be non-zero;

Note 2: uC resets (via WDT trap) if ticker - last_transmitted > 20 minutes. Same for last_received.
